Claude

AIって難しそう？大丈夫です。このページを読めば、Claudeがどんなものか、何ができるのかがまるごとわかります。

Claudeのことを、ここで全部わかるようにしました。

ClaudeはAnthropic（アンソロピック）という会社が作ったAIアシスタントです。「AIって何ができるの？」「使い方がよくわからない」という方のために、専門用語を使わず、わかりやすく解説します。

チャットで質問に答えてもらうだけでなく、パソコンの作業を丸ごと代わりにやってもらったり、ExcelやPowerPointの中から直接使ったりと、使い方はどんどん広がっています。

Claudeとは01

「賢いだけ」じゃなく、「信頼できる」AIを目指している。

世の中にはたくさんのAIがありますが、Claudeを作っているAnthropicという会社は少し違う考え方をしています。「AIは賢ければいい」ではなく、「賢くて、かつ安全で、信頼できるAIを作る」ということを最大の目標にしています。

Claudeは、嘘をつかない・危険なことには協力しない・わからないことは「わかりません」と正直に言う、という姿勢を大切にしています。

💡 たとえるなら… 賢くて正直な「友人」のようなAIです。なんでも「はい、できます！」と言うのではなく、できないことはできないと伝え、正確な情報を届けようとします。

🏛️

Anthropicってどんな会社？

2021年に設立されたAI企業。「AIを安全に発展させること」を最大のミッションとしています。

ポイント
  • AIの安全性を研究する専門家が集まって作った会社
  • 「速く広める」より「安全に作る」を優先している
📜

Constitutional AI（CAI）って何？

Claudeが「なぜそれがダメなのか」を理解して行動するための、Anthropic独自のトレーニング方法です。

わかりやすく言うと
  • ルールを覚えさせるのではなく「なぜそのルールがあるか」を理解させる
  • 人間の道徳教育に近いアプローチで訓練されている
🌐

どうやって使うの？

いくつかの使い方があり、目的によって最適な方法が変わります。

選び方の目安
  • まず試したい → ブラウザで Claude.ai にアクセスするだけ
  • プログラム開発に使いたい → Claude Code
  • 仕事をまるごと任せたい → Claude Cowork
📈

どれくらい使われているの？

2026年初頭の時点で、開発者向けの「Claude Code」だけで年間2,500億円以上の売上を記録するほど急成長中です。

規模感
  • 企業での採用も急増し、わずか8ヶ月で売上が約5倍になった
  • 世界中の企業・個人が業務で使い始めている
Claudeの性格02

Claudeには「人格」がある。

Claudeには、設計として持たせた「性格」があります。フレンドリーでも馴れ馴れしくなく、頭がいいけど偉そうじゃない、丁寧だけどおべっかを使わない——そんな「ちょうどいい」対話相手として設計されています。

💡 たとえるなら… 「なんでも褒めてくれるイエスマン」ではなく、「正直に言ってくれる賢い友人」のイメージです。間違いは間違いと指摘し、わからないことはわからないと伝えます。

好奇心旺盛

どんな話題にも真剣に向き合います。料理の話でも、難しい法律の話でも、Claudeは興味を持って一緒に考えようとします。

率直で正直

知らないことは「知りません」、自信がないことは「これは推測ですが」と伝えます。あなたに気に入られるより、正確な情報を届けることを大切にしています。

共感と配慮

悩みを話しても、一方的に解決策を押しつけません。まずあなたの気持ちに寄り添い、それから一緒に考えてくれます。

場に合ったユーモア

真面目な場面では真面目に、気軽な雑談では少し軽くなります。場の空気を読んで話し方を変えてくれます。

会話の中でよく感じること

Claudeらしい言葉の使い方
  • 「確信はありませんが、おそらく〜だと思います」——不確かなことは必ず断りを入れる
  • 「少し別の視点からも考えてみましょうか？」——反論するときも穏やかに
  • 相手が専門家ならば専門用語で、初心者ならばかみ砕いた言葉で話す
  • 必要な長さだけ答える。長く書けばいいとは思っていない
もっと上手く使うコツ
  • 「遠慮なく批判的なフィードバックをください」と伝えると、本音の改善点を引き出せる
  • 「私は〇〇の専門家です」「私は初心者です」と自己紹介すると、最初から最適なレベルで話してくれる
  • 「私の主張の弱点を指摘してください」と頼むと、自分の考えを強くするために使える
  • 感情的なことを相談するときは「まず話を聞いてほしい」と最初に伝えると、解決策より共感を優先してくれる
モデルの種類03

Claudeには3つのグレードがある。

Claudeは「Opus（オーパス）」「Sonnet（ソネット）」「Haiku（ハイク）」という3つのグレードがあります。賢さとスピードと料金のバランスが異なり、使い道によって最適なものが変わります。

💡 たとえるなら… タクシーに「普通・ハイヤー・高級リムジン」があるようなイメージです。目的地までの移動なら普通で十分、大切な商談なら高級にする、という選び方ができます。

Claude Opus
最も賢い・高機能

難しい問題でも深く考えます。長い文章の分析や、複雑な判断が必要なときに向いています。

最新: Claude Opus 4.6
向いている使い方
  • 難解な契約書や論文を丁寧に読み解く
  • 戦略の立案や複雑な意思決定の補助
  • 長い会話や大量の資料を扱う作業
応用
  • 50本の論文を一度に読み込ませて「この分野の最新トレンドを教えて」と聞く
  • 複数の事業計画を比較して「リスクとメリットを整理して」と依頼する
Claude Sonnet
バランス型・日常使いに最適

賢さとスピードのバランスが良く、ほとんどの日常業務に対応できます。迷ったらこれを選ぶのがおすすめ。

最新: Claude Sonnet 4.6
向いている使い方
  • メール・資料・報告書の作成
  • 調べ物の要約やデータの整理
  • 日常の相談や質問への回答
応用
  • 毎日のルーティン業務（議事録まとめ・メール下書き）を任せる
  • Claude Codeで使う場合も、このモデルが標準でほとんどの開発作業をこなす
Claude Haiku
高速・低コスト・大量処理向け

とにかく速くて安い。単純な作業を大量にこなしたいときに向いています。

最新: Claude Haiku 4.5
向いている使い方
  • お客様サポートのチャットボット
  • 大量の文章を一括で分類・タグ付け
  • シンプルな質問への即答
応用
  • Haikuで一次スクリーニングをして、難しい問い合わせだけOpusに転送する2段構成
  • 1,000件のアンケート回答をHaikuで一括分類して、傾向をまとめる

Claude Code05

プログラマーの「相棒AI」。でも使うのはプログラマーだけじゃない。一般公開

Claude Codeは、パソコンのプログラム開発を手伝ってくれるAIツールです。2025年2月から使えるようになり、2026年初頭には年間2,500億円以上の売上を生む大ヒット製品になりました。

プログラマー向けに作られましたが、最近はプログラムが書けない人が「自分専用の道具」を自分で作るためにも使われています（これを「バイブコーディング」と呼びます）。

💡 たとえるなら… 「何でもできる優秀な助手」がパソコンの中にいて、指示を出すと一人でプログラムを書いたり直したりしてくれます。人間は「何を作ってほしいか」を伝えるだけでOKです。

Claude Code

指示するだけで、
プログラムが
できあがる。

プログラムを書く・直す・テストする作業を、Claudeが自律的に進めます。プログラムが書けない方でも、「作りたいもの」を言葉で伝えるだけで動くものが出来上がります。

主な機能と活用例

🗂️

プロジェクト全体を理解して作業する

ファイルが数百個あるような大きなプロジェクトでも、全部読んで理解した上で作業します。

活用例
  • 「このシステムの認証の仕組みを全ページで統一して」と頼むと、関係するファイルを全部探して直してくれる
応用
  • 新入社員向けに「このシステムの構造を初心者向けに説明して」と頼み、オンボーディング資料を自動生成
  • 古いシステムを見せて「修正が難しい箇所はどこ？」と技術的な負債を洗い出させる
🤖

自動で判断して作業を進める（Auto Mode）

「安全な作業かどうか」をAIが自動で判断して進めます。危ない操作は自動でストップ。

活用例
  • テストを実行 → 失敗 → 原因を特定 → 修正 → 再テスト、のサイクルを全自動で回す
応用
  • 夜間に「昨日失敗したテストを直して、報告書を作っておいて」と頼んで帰宅。翌朝には完了している
  • 週に一度、使っているソフトウェアの更新と動作確認を自動で行うよう設定する
💬

DiscordやTelegramからも操作できる

スマホのチャットアプリからClaudeに指示を送り、パソコンで作業させることができます。

活用例

応用
  • チームのSlackに「Claudeボット」として設置して、誰でも「コードを確認して」と頼めるようにする
  • 企画担当者や営業担当者が日本語で仕様を伝えると、Claudeが自動でプログラムを作るフローを構築

他のサービスと繋いで使う（MCP）

Figmaのデザインツールや、Jiraというタスク管理ツールなど、外部のサービスと連携して使えます。

  • 「Aさんが担当しているタスク一覧を取得して進捗をまとめて」と頼む
応用
  • デザイン確認→コーディング→テスト→タスク完了報告の流れを一気通貫で自動化

マウスとキーボードで画面を直接操作

Claudeが実際にパソコンの画面を見ながら、マウスをクリックしたりキーを打ったりして作業します。

活用例

応用
  • 「このウェブページが正しく動くか全パターンテストして、問題があればスクリーンショット付きで報告して」と頼む
  • 複数の管理画面を横断しながら設定確認する作業をまとめて自動化
📝

プログラムが書けない人でも使える（バイブコーディング）

「こんなものを作りたい」と日本語で伝えるだけで、Claudeが全部プログラムを書いてくれます。

活用例
  • 「毎月の経費をグラフ表示する簡単なツールを作って」と言うだけで動くものができる
応用
  • 部署ごとに専用の業務ツールを現場の担当者自身が作る「市民開発」の仕組みを整える
  • 「翌日の会議でデモが必要」というときに、当日中にプロトタイプを作って見せる
```

CLAUDE.md — 「Claudeへの業務マニュアル」ファイル

CLAUDE.mdとは、テキストで書いた「Claudeへの取扱説明書」です。このファイルをフォルダに置いておくと、Claude Codeを起動するたびに自動で読み込まれ、「このプロジェクトではどう動けばいいか」をClaudeが把握してから作業を始めます。

💡 たとえるなら… 新しいアルバイトに「うちの店のルール」を書いた紙を渡すイメージ。毎回口頭で説明しなくても、Claudeはそのルールを守って動いてくれます。

📂 どこに置くかで効果が変わる

CLAUDE.mdは複数の場所に置けます。置く場所によって「どの範囲に適用されるか」が変わり、近い場所のルールが優先されます。

# フォルダの構成イメージ
ホームフォルダ（自分のパソコン全体）
└── CLAUDE.md ① 全プロジェクト共通のルール

営業ツール/（特定の仕事フォルダ）
├── CLAUDE.md ② この仕事専用のルール
│
├── ソースコード/
│ └── CLAUDE.md ③ このフォルダだけのルール
│
└── ドキュメント/
　　└── CLAUDE.md ③ このフォルダだけのルール
①
全共通
ホームフォルダに置く。「私はいつも日本語で答えて」「説明は短めにして」など、どの仕事でも共通の自分ルールを書く。
②
仕事別
仕事フォルダのトップに置く。「この仕事はPythonで」「テストは必ず書いて」などこの仕事全体のルール。チームで共有できる。
③
場所別
特定のフォルダ内に置く。「ここのファイルは触る前に必ず確認して」などそのフォルダだけの細かいルール。②のルールより優先される。

読み込み順：① → ② → ③ の順に読まれ、後から読んだものが優先されます。

✏️ 実際の書き方の例（そのままコピーして使えます）

# この仕事について
営業チーム向けの顧客管理ツールです。

# 言葉のルール
- コードのコメントと説明は必ず日本語で書いてください
- 難しい用語は使わず、わかりやすく説明してください

# 作業のルール
- 新しい機能を追加したら、必ずテストも書いてください
- 大きな変更の前に、「こうしようとしています」と教えてください

# ⚠️ 絶対にやらないでほしいこと
- お客様の個人情報を外部に送らないでください
- 本番のデータベースに直接書き込まないでください
活用例
  • コーディングのルールを書いておいて、毎回言わなくても自動で守らせる
  • 「やってはいけないこと」を明示して、Claudeが誤って重要なファイルを触らないようにする
  • 「変更前に必ず報告して」と書いて、不意の書き換えを防ぐ安全装置として使う
応用
```

安全性への姿勢10

「安全」は後付けじゃなく、最初から設計に入っている。

Anthropicは「AIの安全性」を最も大切なテーマとして研究している会社です。Claudeは「危ないことには手を貸さない」「嘘をつかない」「本当に役に立つ」という3つを基本の考え方として作られています。

💡 たとえるなら… 優秀な医師は患者に「この薬を増量して」と言われても、危険なら断ります。Claudeも同じで、「やってほしいこと」が危険だと判断した場合は正直に断り、その理由を伝えます。

「ルールを覚えさせるのではなく、なぜそのルールがあるのかを理解させる。そのために憲法（Constitution）を作り、AIに自ら学ばせた。」

— Anthropic Research
I

危害を加えない（Harmlessness）

人や社会を傷つける可能性のある情報や内容の作成を断ります。「危ない」と判断したら、たとえ相手に頼まれても協力しません。

II

正直に話す（Honesty）

知らないことを知ったかぶりしません。自信がないときは「これは推測ですが」と必ず断ります。Claudeは自分がAIであることも隠しません。

III

本当に役立つ（Helpfulness）

表面的な要求だけでなく、「この人が本当に必要としているもの」を考えて答えます。求めていることと必要なことが違うと思ったら、そのことも伝えます。

IV

Constitutional AI（CAI）とは

AIに「どう行動するか」のルール集（憲法）を与えて、自分でそのルールに照らして自分の答えを評価・改善させる訓練方法です。人間が全部チェックするのではなく、AIが自律的に学ぶ仕組みです。

V

安全レベルの格付け制度

Anthropicはモデルの危険度をレベル1〜4で評価しています。Claude Opus 4はレベル3（高リスク）に分類されており、より慎重な管理のもとで運用されています。

VI

人間が最終判断をする（Human Oversight）

AIがどれだけ賢くなっても、重要な決定は人間が行うべきだという考えを大切にしています。Claudeはその姿勢を支持する設計になっています。
